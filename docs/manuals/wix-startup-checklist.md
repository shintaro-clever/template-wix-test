# Wix Startup Checklist

## Phase 1：技術セットアップ（管理者担当）

### Step 1：Wix 生成リポジトリの用意
- 確認する: Wix Studio GitHub Integration で実働先リポジトリを生成した（実働先は Wix 生成リポジトリ）
- 確認する: GitHub 上にリポジトリが存在し、Wix がプッシュしたファイルが含まれている
- 触らない: テンプレート基盤を実働先として使わない（Wix は自身が生成したリポジトリしか認識しない）

### Step 2：wix.config.json の確認
- 確認する: `wix.config.json` がリポジトリ直下に存在する
- 確認する: `siteId` が Wix 生成の値で設定されている
- 確認する: `uiVersion: "6"` が含まれている（ない場合は手動追記してコミットする）
- 触らない: `siteId` をテンプレート基盤側の値で上書きしない

### Step 3：src/ の存在確認
- 確認する: `src/pages/masterPage.js` が存在する（Wix Studio GitHub Integration が自動生成）
- 確認する: `src/backend/permissions.json` が存在する
- 確認する: `src/styles/global.css` が存在する
- 触らない: `src/` をテンプレート基盤のファイルで上書きしない（Wix 生成構造を維持する）

### Step 4：WIX_API_KEY の設定
- 確認する: Wix ダッシュボードで API キーを発行済み
- 確認する: 生成リポジトリの Settings → Secrets → Actions に `WIX_API_KEY` が設定されている

### Step 5：CI 動作確認（wix preview --source remote）
- 確認する: `.github/workflows/wix-preview-on-push.yml` が存在する
- 確認する: `main` push 後に GitHub Actions `wix-preview-on-push` が SUCCESS になる
- 確認する: Actions ログにプレビュー URL（`https://wix.to/...`）が出力されている
- 触らない: CI が失敗している状態のまま次のステップに進まない
- 覚える: このワークフローはプレビュー URL を生成するのみで、本番公開は行わない

### Step 6：テンプレート基盤からの資産移植確認
- 確認する: `.github/workflows/` に CI ワークフローが移植されている
- 確認する: `agents/`、`docs/`、`scripts/`、`prototype/` が移植されている
- 確認する: `AGENTS.md`、`CLAUDE.md`、`README.md` が存在する
- 確認する: `package.json` に `@wix/cli` devDependency が追記されている
- 触らない: 移植時に `src/` や `wix.config.json` を既存側の内容で上書きしない

## Phase 2：最小検証（非エンジニアも参加）

### 開始前
- 確認する: 案件名と対象サイトが決まっている
- 確認する: Wix Studio の対象案件に入れる
- 確認する: 使う文言、画像、ロゴの有無が分かっている
- 確認する: 最終調整を誰が担当するか決まっている
- 確認する: 今回は本番全体ではなく最小検証から始めると共有されている
- 依頼する: 権限が不足している場合は管理者に追加を依頼する
- 触らない: Git や CLI の詳細操作を自己判断で始めない

### 開始直後
- 確認する: まず何を検証するかが1ページまたは1セクションに絞れている
- 確認する: 静的原型（`prototype/minimum-page/`）の置き場が分かっている
- 確認する: 自分が進める範囲と管理者確認が必要な範囲が分かっている
- 依頼する: 検証対象が決めきれない場合は管理者に相談する
- 依頼する: 素材や文言が足りない場合は関係者に依頼する
- 触らない: 本番ページ全体の作り込みを先に始めない

### 最小検証前
- 確認する: Hero、説明、CTA、画像領域など最低限の構成が決まっている
- 確認する: テキスト差し替え、画像差し替え、CTA変更を試す前提になっている
- 確認する: 最小検証の結果で Go / No-Go を判断する前提が共有されている
- 依頼する: 接続や構造変更で迷ったら管理者に確認する
- 依頼する: Wix Studio だけで吸収できない変更は技術担当へ相談する
- 触らない: 検証前に本番公開前提の最終仕上げへ進まない

## Phase 3：本制作・引継ぎ前

- 確認する: 最小検証で分かったことが整理されている
- 確認する: 通常更新を Wix Studio 側で回せる見込みがある
- 確認する: `main` push → プレビュー URL 自動生成のフローを関係者が把握している
- 確認する: 誰が次の作業を担当するか明確になっている
- 依頼する: 本制作へ進む判断が必要なら管理者へ確認する
- 依頼する: 方針修正が必要なら判断材料をまとめて共有する
- 触らない: 判断未了のまま本制作へ広げない
