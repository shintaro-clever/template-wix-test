# Go / No-Go Criteria

## 目的
- 最小検証ページの結果をもとに、本制作へ進むか、方針を修正するかを判断する
- 感覚ではなく、編集性、再現性、運用負荷の観点で次アクションを決める

## 判定観点

### 編集性
- Wix Studio 上で見出し、本文、画像、CTA を個別に編集できるか
- 編集時に他要素まで巻き込んで崩れにくいか

### 再現性
- 静的原型の構造を Wix Studio 上で無理なく再現できるか
- 最小検証の結果を、次のページやセクションへ横展開できそうか

### 差し替えやすさ
- テキスト差し替えが短時間で完了するか
- 画像差し替えや CTA 変更で構造が壊れないか
- セクション順変更が実運用で許容できる手間で済むか

### 担当者の運用負荷
- 非エンジニア担当者でも Wix Studio 上で通常更新を回せるか
- 日常更新のたびに repo やコード修正へ戻らなくて済むか

### Wix Studio 側での最終調整容易性
- 余白、配置、見た目の微調整を Wix Studio 上で完結できるか
- 最終仕上げのために別途コード改修が常態化しないか

## Go 条件
- 編集性に大きな問題がない
- 静的原型を Wix Studio 上で十分に再現できる
- テキスト、画像、CTA の差し替えが運用上の負荷にならない
- 非エンジニア担当者でも Wix Studio 側で最終調整と通常更新を行える
- 本制作へ広げても repo 側の追加作業が限定的で済む見込みがある

## 方針修正条件
- 最小検証自体は成立したが、一部の編集や差し替えに手戻りが多い
- セクション構造は再現できるが、運用負荷が想定より高い
- Wix Studio 側だけで完結しない箇所があり、原型や持ち込み方を調整したほうがよい

## No-Go 条件
- 基本編集で構造が崩れ、日常運用に耐えない
- 静的原型を Wix Studio 上で安定して再現できない
- 差し替えのたびにコード修正が必要になりそう
- 非エンジニア担当者が Wix Studio 側で更新を回せない
- 本制作へ進むと、毎回 repo 側へ戻る運用になりそう

## 判定後の進め方

### Go
- 次のページ単位または追加セクションへ展開する
- 最小検証で有効だった構造を基準に本制作へ進む

### 方針修正
- 原型の構造、セクション分け、CTA 配置、画像領域の取り方を見直す
- 必要なら検証対象をさらに小さくして再試行する

### No-Go
- 本制作へ進まない
- Wix Studio 前提の作り方自体を見直し、原型の作成方法または運用前提を再設計する
