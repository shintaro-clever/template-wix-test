# Go / No-Go Criteria

## 目的
- 最小検証ページの結果をもとに、本制作へ進むか、方針を修正するかを判断する
- 感覚ではなく、編集性、再現性、運用負荷の観点で次アクションを決める
- 次回の再試行対象をぶらさず、同じ `Hero` と同じ `site ID` 条件で判定できるようにする
- 現時点の阻害要因が prototype 欠陥ではなく、認証未到達であることを前提に判定する

## 固定前提
- 次回の再実施対象は引き続き `Hero`
- 対象 `site ID` は `0e9fab77-6694-464e-9f13-d5c320c88550`
- 別セクション、別ページ、本番全体への対象拡大は行わない
- まず同一条件で再試行し、その結果をもとに判定する

## 現フェーズの完了条件
- 現フェーズの完了条件は、対象 `site ID 0e9fab77-6694-464e-9f13-d5c320c88550` の案件画面が表示されること
- 案件画面到達前は、`Hero` 再構成、編集性確認、最終 `Go / No-Go` 実判定に進まない
- 案件画面に到達できた場合のみ、次に `WIX-STUDIO-14` → `WIX-STUDIO-15` → `WIX-STUDIO-16` に進む
- したがって、この文書の `Go / No-Go` 判定枠は現フェーズの主タスクではなく、後続条件タスクの受け皿として保持する

## 現在の暫定判定の意味
- 現時点の `方針修正` は、prototype 欠陥を意味しない
- 現時点の No-Go 相当理由は、Wix Studio の案件画面まで到達できていないことにある
- 現在の `Fail` は認証未到達による停止であり、prototype 欠陥判定ではない
- したがって、認証導線が固定されるまでは、prototype 自体の可否を最終判定しない
- 次回は、認証通過後の実施結果をもとに、暫定判定から実判定へ移る
- `site ID 0e9fab77-6694-464e-9f13-d5c320c88550` は確定済みで、`prototype/minimum-page/` は開始条件を満たしている
- `https://manage.wix.com/studio` と対象 dashboard URL は、3 回目の再確認でもどちらも `Sign up / Log in` 画面だった
- `WIX-STUDIO-14` と `WIX-STUDIO-15` は、案件画面到達後にのみ開始する

## 判定観点

### 編集性
- Wix Studio 上で見出し、本文、画像、CTA を個別に編集できるか
- 編集時に他要素まで巻き込んで崩れにくいか

### 再現性
- 静的原型の構造を Wix Studio 上で無理なく再現できるか
- 最小検証の結果を、次のページやセクションへ横展開できそうか

### 差し替えやすさ
- テキスト差し替えが短時間で完了するか
- 画像差し替えや CTA 変更で構造が壊れないか
- セクション順変更が実運用で許容できる手間で済むか

### 担当者の運用負荷
- 非エンジニア担当者でも Wix Studio 上で通常更新を回せるか
- 日常更新のたびに repo やコード修正へ戻らなくて済むか

### Wix Studio 側での最終調整容易性
- 余白、配置、見た目の微調整を Wix Studio 上で完結できるか
- 最終仕上げのために別途コード改修が常態化しないか

## Go 条件
- 対象 `site ID` の案件画面まで到達できる
- `Hero` の再構成が完了する
- 主要編集操作として、テキスト差し替え、画像差し替え、CTA 変更を確認できる
- セクション単位の扱いやすさと順序変更の確認ができる
- 非エンジニア担当者でも Wix Studio 側で通常更新を回せる見込みがある
- 本制作へ広げても repo 側の追加作業が限定的で済む見込みがある

## 方針修正条件
- 案件画面には到達できたが、`Hero` の再構成または主要編集操作の一部で手戻りが多い
- 最小検証自体は成立したが、一部の編集や差し替えに手戻りが多い
- セクション構造は再現できるが、運用負荷が想定より高い
- Wix Studio 側だけで完結しない箇所があり、原型や持ち込み方を調整したほうがよい
- `Hero` と同じ `site ID` 条件で再試行する前に、別対象へ広げるべきではないと判断される

## No-Go 条件
- 認証導線が固定された後でも、対象 `site ID` の案件画面へ安定して到達できない
- 基本編集で構造が崩れ、日常運用に耐えない
- 静的原型を Wix Studio 上で安定して再現できない
- 差し替えのたびにコード修正が必要になりそう
- 非エンジニア担当者が Wix Studio 側で更新を回せない
- 本制作へ進むと、毎回 repo 側へ戻る運用になりそう

## 実判定へ移る条件
- 認証が通り、対象 `site ID` の案件画面まで到達している
- `Hero` の再構成を実際に完了している
- 主要編集操作として、テキスト差し替え、画像差し替え、CTA 変更を確認している
- セクション単位の扱いやすさ、順序変更、非エンジニア運用性を確認している
- 以上を満たした時点で、`Go / 方針修正 / No-Go` を暫定ではなく実判定に切り替える

## 判定後の進め方

### Go
- 次のページ単位または追加セクションへ展開する
- 最小検証で有効だった構造を基準に本制作へ進む

### 方針修正
- まず同じ `Hero` と同じ `site ID` 条件で再試行する
- 原型の構造、セクション分け、CTA 配置、画像領域の取り方を見直す
- 必要なら検証対象をさらに小さくして再試行する
- 別対象へ拡大して問題をぼかさない

### No-Go
- 本制作へ進まない
- Wix Studio 前提の作り方自体を見直し、原型の作成方法または運用前提を再設計する
