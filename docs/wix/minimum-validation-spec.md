# Minimum Validation Spec

## 目的
- Wix Studio 連携の初回検証対象を最小単位で固定し、いきなり本番 LP 全体へ進まないようにする
- 外部 AI 制作物を Wix Studio へ持ち込んだときの編集性、再編集性、運用しやすさを先に確認する
- 次回の再開は prototype 修正ではなく、Studio ブラウザの認証導線確認から始める

## 検証対象
- 本番全体ではなく、1ページまたは1セクション相当の最小単位を対象にする
- 想定対象はヒーロー、紹介セクション、問い合わせ導線などのうち1つ
- 初回検証候補は Hero または問い合わせ導線のような、差し替え点が分かりやすい単位を優先する
- AI で作った最小 UI を Wix Studio に持ち込み、編集可能な形で再調整できるかを見る
- 次回の再実施対象も引き続き `Hero` に固定する
- 対象を別セクションや本番全体へ拡大せず、同一条件で再試行する

## 再開時の手順順
1. Studio ブラウザでログイン状態を確認する
2. 対象 URL へ到達できることを確認する
3. 案件画面が表示されることを確認する
4. その後に Hero の再構成を始める

## 案件画面到達後の最小手順
1. 対象は `Hero` のみに固定し、別セクションへ広げない
2. 見出し、本文、画像、CTA の 4 要素で Hero の骨組みを置く
3. 見出しと本文を差し替え、画像を 1 回差し替える
4. CTA 文言とリンク先を変更する
5. Hero 単位で選択と再調整ができるか確認し、編集性確認へ進む

## 対象案件の固定情報
- 対象 URL は `https://manage.wix.com/dashboard/0e9fab77-6694-464e-9f13-d5c320c88550/setup?referralInfo=my-sites`
- 対象 `site ID` は `0e9fab77-6694-464e-9f13-d5c320c88550`
- 次回の再実施も上記 `site ID` を前提条件とする

## 非対象
- 本番 LP 全体の作成
- 全ページ展開
- 別セクションや別ページへ対象を広げること
- 本番用の文言、画像、導線、SEO の最終確定
- 高度なアニメーションや複雑な CMS 連携の先行実装
- Studio ブラウザへ入る前に prototype の再修正を始めること

## 確認観点
- Studio ブラウザで案件画面まで入れるか
- Wix Studio 上でレイアウト、テキスト、画像差し替えが無理なくできるか
- 外部 AI 制作物を持ち込んだ後も、Wix Studio 側で再編集しやすい構造になっているか
- セクション差し替えや軽微修正を案件運用で回せるか
- Git / Codespaces 側の作業結果と Wix Studio 側の編集フローが衝突しないか

## 合格条件
- Studio ブラウザで対象案件画面まで入れる
- 最小対象を Wix Studio に持ち込み、表示確認ができる
- テキスト修正、画像差し替え、余白調整などの基本編集が Wix Studio 上で完了する
- 再編集時に構造崩れや運用上の大きな手戻りが出ない
- 次にどの構成で横展開するかを判断できるだけの知見が残る

## 次段階への移行条件
- 認証導線が固定され、同じ対象 URL で再開できる
- 同じ `site ID` と同じ `Hero` 条件で再試行した結果が整理されている
- 最小単位での編集性と再編集性に問題がない
- 運用上の詰まりどころが整理されている
- 次にページ単位へ広げるか、別セクションを追加検証するかを決められる
