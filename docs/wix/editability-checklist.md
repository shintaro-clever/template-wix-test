# Editability Checklist

## 目的
- Wix Studio へ移した後に、最小ページが再編集しやすい構造になっているかを確認する
- テキスト差し替え、画像差し替え、セクション順変更、CTA 変更を先に観点化する
- Wix Studio の認証が通った直後に、そのまま Hero の最小検証へ移れるようにする
- 現フェーズでは主タスクとして扱わず、`WIX-STUDIO-14` 完了後の後続条件タスクとして使う

## 認証通過直後の実施順
1. Hero セクション単位で選択できるか確認する
2. テキスト差し替えを試す
3. 画像差し替えを試す
4. CTA の変更を試す
5. 順序変更やセクション単位の扱いやすさを確認する
6. 非エンジニア運用として回せるか判断する

## 記録形式
- 各項目は `Pass / Partial / Fail` のいずれかで残す
- 各項目には、一言で根拠を書く
- 長い説明は避け、比較できる粒度で揃える

### 記入テンプレート
- テキスト差し替え: `Pass | Partial | Fail` - 根拠
- 画像差し替え: `Pass | Partial | Fail` - 根拠

- 順序変更: `Pass | Partial | Fail` - 根拠

## Hero 向け確認項目

### 1. セクション単位の扱いやすさ
- Hero 全体を 1 セクションとして把握できる
- Hero 内の見出し、本文、画像、CTA を個別に選択できる
- Hero を触っても他セクションへ影響が広がりすぎない
- Hero 単位で再配置や調整を考えやすい

### 2. テキスト差し替え
- Hero の見出しを差し替えられる
- Hero の本文を見出しと分けて差し替えられる
- テキスト差し替え後に配置や余白が大きく崩れない
- 非エンジニア担当者でも、どこを直すか迷いにくい

### 3. 画像差し替え
- Hero の画像領域を独立して選択できる
- 画像を差し替えても隣接テキストの構造を壊さない
- 画像サイズや見え方の調整が Hero 内で完結する
- 画像差し替えのたびにコード側へ戻る必要がない

### 4. CTA 変更
- Hero の CTA 文言を変更できる
- CTA のリンク先を独立して変更できる
- 主要 CTA と補助 CTA を見分けやすい
- CTA 変更後も Hero 全体のバランスが崩れにくい

### 5. 順序変更
- Hero と他セクションの順序変更を試せる
- 順序変更時に Hero 内の構造が壊れない
- 順序変更が実運用で許容できる手間で済む

### 6. 非エンジニア運用の成立性
- 日常更新で必要な操作が Wix Studio 上で完結する
- 「どこを触ってよいか」が画面上で判断しやすい
- 軽微修正のたびに repo やコード修正へ戻らなくて済む
- 非エンジニア担当者でも Hero の更新を引き継げる見込みがある

## 確認観点

### テキスト差し替え
- 各セクションの見出しと本文が分かれている
- 見出し階層が `h1 -> h2 -> h3` の順で整理されている
- 本文差し替え箇所がまとまっており、他要素に巻き込まれにくい

### 画像差し替え
- 画像領域が独立したブロックになっている
- 画像差し替え時に隣接テキストの構造を壊さずに済む
- 画像サイズ変更の影響範囲がセクション内で閉じている

### セクション順変更
- セクション境界が明確で、役割ごとに分かれている
- Hero、Overview、Explanation、CTA の順を入れ替えても意味が崩れにくい
- セクション内の要素が他セクションに依存していない

### CTA 変更
- CTA 文言の差し替え箇所が明確
- CTA リンク先を独立して変更できる
- 主要 CTA と補助 CTA が視覚的に区別されている

## 最小ページで見るポイント
- `prototype/minimum-page/index.html` に section ごとの境界と役割ラベルがある
- 見出し、本文、CTA、画像プレースホルダが分離されている
- Wix Studio 上で一部差し替え後も、再編集と再配置が続けられる構造になっている
- 認証通過後は、まず Hero の確認から始め、別対象へ広げない
