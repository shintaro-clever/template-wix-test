# /design-review コマンド

## 概要

対象ファイル（HTMLモックアップ / CSS / Veloコード）を `agents/rules/60-design-quality.md` のチェックリストで評価し、レポートを出力する。

## 使い方

```
/design-review <ファイルパス>
/design-review src/mockups/green-drive-top.html
/design-review src/styles/global.css
```

引数なしの場合は `src/mockups/` 配下の最新ファイルを対象にする。

---

## Claude の実行手順

### Step 1: ファイル読み込み

対象ファイルを Read する。CSSの場合は `src/styles/global.css` も合わせて読む。

### Step 2: 自動検証（数値チェック）

`scripts/design-check.js` を実行して CSS token 値の数値検証を先に行う：

```bash
node scripts/design-check.js
```

### Step 3: AIレビュー（チェックリスト評価）

以下のカテゴリをコードの内容から評価し、各項目を `✅ / ⚠️ / ❌ / 確認不可` で判定する：

| 記号 | 意味 |
|---|---|
| ✅ | 基準を満たしている |
| ⚠️ | 改善が望ましい |
| ❌ | 基準を満たしていない（要修正） |
| 確認不可 | コードからは判断できない（人間が確認） |

評価カテゴリ：
1. レスポンシブ基準（フォントサイズ・マージン・ブレイクポイント）
2. ビジュアル（カラム構成・ファーストビュー・余白）
3. カラー配色（使用色数・コントラスト比・トンマナ）
4. フォント（可読性・ブランド適合）
5. UI要素（ヘッダー・フッター・ボタン・インプット）
6. アニメーション（過剰でないか）
7. 機能設計（セキュリティ・UX）

### Step 4: レポート出力

以下のフォーマットで出力する：

```
## デザインレビューレポート
対象: <ファイルパス>
日時: <日付>

### サマリー
- ✅ 合格: X 項目
- ⚠️ 要改善: X 項目
- ❌ 不合格: X 項目
- 確認不可: X 項目

### 詳細

#### レスポンシブ基準
- ✅ フォントサイズ（H1: 36px — 基準: 30〜40px）
- ❌ スマートフォンのサイドマージン（8px — 基準: 16px 以上）

...（カテゴリ別）

### 修正提案
❌ / ⚠️ 項目について具体的な修正コードを提示する。
```

### Step 5: 修正の実施

ユーザーが「修正して」と言った場合、❌項目から優先的に修正する。
修正後は再度 Step 2〜4 を実行して合格を確認する。
