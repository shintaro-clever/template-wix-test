# デザインチェックシート

ソース: `docs/quality/design-checklist.csv`

Figma → Wix移行時、コードレビュー時、デザイン評価時に以下の基準を参照する。

---

## ビジュアル

- [ ] 1カラムだけではなく、2〜4カラムを適所に配置して、ページ全体に動きをつけているか
- [ ] ファーストビューは100vhが適正か
  *備考：背景や要素が質素で間延びしていたり、以下コンテンツの情報が重要である場合は最小限の縦幅の方が良い*
- [ ] ページ全体として、バランス良い余白や素材配置はなされているか

---

## カラー配色

- [ ] デザイン内で使用するメインカラー・キーカラーは3種類程度に抑えているか
  *備考：文字色や背景色などは除く*
- [ ] 特に視認性が必要な箇所についてのコントラスト比が4.5:1になっているか
- [ ] ブランドやサービスに合ったカラーリングを選定しているか
  *備考：適切なトンマナを設定できているか*
- [ ] 特別な場合を除き、カラー展開の彩度や明度が統一されたものであるか

---

## フォント

- [ ] サイトをリーチさせたいターゲット層が視認できるフォントを選定しているか
  *備考：一部の明朝体やローマン体などは、文字の中で細い部分があるため*
- [ ] ブランドやサービスに合ったフォントを選定しているか
- [ ] フォントに縁をつけたり、枠をつけたりした場合の、視認性や可読性は担保されているか

---

## アイコン

- [ ] あらゆるユーザーが理解できるユニバーサルデザインのアイコンを選定しているか
- [ ] アイコンと文字情報の内容が連動しているか
- [ ] トンマナに則したアイコンデザインになっているか

---

## シェイプ

- [ ] 余白への収まりや全体のレイアウトとのバランスを考慮して、整形や処理を行なっているか
- [ ] シェイプの上に文字情報を載せている場合、文字情報を邪魔しない形に抑えているか
- [ ] ブランドやサービスに合った形になっているか
  *備考：スマートなコーポレートサイトなどで個性的な形になりすぎていないか*

---

## パターン（模様）

- [ ] セクションやコンテンツ、タイトルバーの情報を邪魔していない濃度の模様になっているか
- [ ] 適材適所なパターン配置になっているか
  *備考：雰囲気や動きを変える目的でパターン（模様）を配置するので、単色・グラデーション背景とのバランスが大切*
- [ ] トンマナに則したパターンになっているか

---

## 画像

- [ ] トンマナに則したコントラストや色調になっているか
- [ ] フレームやシェイプを施している場合、トンマナに則した形状やデコレーションをしているか
- [ ] ファイルサイズや解像度が、サーバー容量やサイトの表示スピードを逼迫させない範囲であるか
- [ ] フリー素材を使用する場合、素材の著作権や肖像権、商用利用の可否について確認はされているか

---

## イラスト

- [ ] トンマナに則したイラストデザインになっているか
- [ ] フリー素材を使用する場合、素材の著作権や商用利用の可否について確認はされているか
- [ ] 手書き風のイラストの場合、塗りがはみ出ていたり、線が滑らかであるか
- [ ] イラストが画像ファイルの場合、一般的な解像度が保たれているか

---

## 動画

- [ ] トンマナに則したコントラストや色調になっているか
- [ ] フレームやシェイプを施している場合、トンマナに則した形状やデコレーションをしているか
- [ ] ファイルサイズや再生時間、解像度が、サーバー容量やサイトの表示スピードを逼迫させない範囲であるか
- [ ] フリー素材を使用する場合、素材の著作権や肖像権、商用利用の可否について確認はされているか

---

## BGM

- [ ] サイト全体の雰囲気にあったBGMを設定できているか
- [ ] フリー素材を使用する場合、素材の著作権や商用利用の可否について確認はされているか
- [ ] 再生される音量は適正か
  *備考：デフォルトの状態が、ミュートでなくても良いか要確認*

---

## ヘッダーメニュー

- [ ] トップに遷移するロゴ画像やお問い合わせボタン、ハンバーガーボタンなどを適切な大きさで配置しているか
  *備考：適切なマージンで適所に配置されているか・クリックしやすく、文字情報は視認性が高いか*
- [ ] テキストメニューの可読性が高く、必要に応じてホバーや選択中ステートを設定しているか
- [ ] テキストメニューの横幅を考慮して、適切な画面サイズでハンバーガーメニューに切り替えているか

---

## フッターメニュー

- [ ] トップに遷移するロゴ画像やSNSボタンなどを適切な大きさで配置しているか
- [ ] テキストメニューの可読性が高く、必要に応じてホバーや選択中ステートを設定しているか
- [ ] テキストメニューの横幅を考慮して、適切な改行や配置をしているか

---

## サイドメニュー

- [ ] テキストメニューの可読性が高く、必要に応じてホバーや選択中ステートを設定しているか
- [ ] メインコンテンツを侵害しない横幅で設置できているか
- [ ] メインコンテンツの縦幅に応じて、ページスクロールに追従しているか

---

## タブメニュー

- [ ] テキストメニューの可読性が高く、必要に応じてホバーや選択中ステートを設定しているか
- [ ] テキストメニューの横幅に応じて、横スクロールできているか
- [ ] 必要に応じて、ページスクロールに追従しているか

---

## ボタン

- [ ] 余白への収まりや全体のレイアウトとのバランスを考慮して、整形や処理を行なっているか
- [ ] 視認性や可読性を考慮したテキストサイズやテキストウェイトになっているか
- [ ] シェイプの上に文字情報を載せている場合、文字情報を邪魔しない形に抑えているか
- [ ] ブランドやサービスに合った形になっているか

---

## インプット要素

- [ ] 入力・クリックしやすい領域を確保しているか
- [ ] 必要に応じて、プレースホルダーを設置し、入力文字を侵害しない色の薄さになっているか
- [ ] 必要に応じて、各ステートごとにデザインパターンを作成しているか

---

## 機能設計

- [ ] ユーザーや管理者のユースケースに合わせた設計になっているか
- [ ] 実装時に実現可能なシステムやUI/UXを考慮した設計になっているか
- [ ] セキュリティ要件を満たした設計になっているか

---

## アニメーション

- [ ] トンマナやデザインガイドラインから逸脱した（または、デザインを崩壊させる）アニメーションを設定していないか
- [ ] ユーザーに疲労感や嫌悪感を与えるアニメーションを設定していないか
- [ ] 画面の読み込み速度やサーバー・インフラに支障をきたすほどのアニメーションを設定していないか

---

## スケジュール進行

- [ ] クライアントもしくはチームリーダーが設定したスケジュールでデザイン設計の進行はできていたか
- [ ] スケジュールの遅延が発生した場合、リカバリーのための作業または提案・交渉はできていたか
- [ ] スケジュールの遅延を予測して、余裕を持ってデザイン設計・提案はできていたか

---

## コミュニケーション

- [ ] クライアントもしくはチームリーダーの意図を汲み取り、適切なデザイン設計に努められたか
- [ ] 不明点・乖離点を適切に伝えて、作業に支障をきたさない状態でデザイン設計に従事できたか
- [ ] チームの輪を乱さずに、相手に配慮・考慮したコミュニケーションを図り、円滑なプロジェクト進行ができたか

---

## Claude Codeでの使い方

この基準は以下の場面で参照する：
1. **Figma → Wix移行時** — Figmaデザインがレイアウト・フォント・カラー基準をクリアしているか確認
2. **CSSレビュー時** — `src/styles/global.css` のトークン値がレスポンシブ基準に沿っているか確認
3. **モックアップ生成時** — 生成したHTMLがフォントサイズ・マージン基準を満たしているか確認
4. **PRレビュー時** — デザイン変更が品質基準を下回っていないか確認
